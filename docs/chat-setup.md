# צ׳אט אונליין — הפעלה

הצ׳אט עובד בשני מצבים אוטומטית:

- **מקומי (ברירת מחדל)** — ללא הגדרות. ההודעות נשמרות ב־`localStorage` של הדפדפן בלבד.
- **אונליין** — משותף, נשמר בדאטהבייס, מתעדכן בזמן אמת ומחייב התחברות. מופעל כשכל משתני הסביבה קיימים.

המעבר אונליין קורה לבד ברגע שהמפתחות מוגדרים ב־`.env` — אין שינוי קוד.

## שכבות

| שכבה | שירות | מה זה נותן |
| --- | --- | --- |
| דאטהבייס | Neon Postgres | שמירת הודעות (טבלה נוצרת אוטומטית) |
| אימות | Clerk | הרשמה/התחברות (אימייל + Google), פרופיל, אווטאר |
| Realtime | Pusher Channels | דחיפת הודעות חדשות לכל המחוברים |

## שלבי הגדרה

1. **Neon** — צרו פרויקט ב־[neon.tech](https://neon.tech), העתיקו את ה־connection string אל `DATABASE_URL`.
2. **Clerk** — צרו אפליקציה ב־[clerk.com](https://clerk.com), הפעילו Email + Google, והעתיקו את `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` ו־`CLERK_SECRET_KEY`.
3. **Pusher** — צרו אפליקציה ב־[pusher.com/channels](https://pusher.com/channels), והעתיקו `PUSHER_APP_ID`, `PUSHER_SECRET`, `NEXT_PUBLIC_PUSHER_KEY`, `NEXT_PUBLIC_PUSHER_CLUSTER`.
4. העתיקו את `.env.example` ל־`.env`, מלאו את הערכים, והריצו מחדש `npm run dev`.

הטבלה `chat_messages` נוצרת אוטומטית בפנייה הראשונה — אין צורך בהרצת מיגרציה ידנית.

## אבטחה ומודרציה שכבר מובנות

- כתיבה עוברת רק דרך `/api/chat/[room]` עם אימות Climb (Clerk) — אי אפשר לכתוב לדאטהבייס מהדפדפן.
- הגבלת קצב: עד 8 הודעות ב־10 שניות למשתמש.
- אורך מקסימלי: 500 תווים; הודעות ריקות נחסמות.
- טקסט מוצג דרך React (escaping מובנה) — הגנה מפני XSS.
